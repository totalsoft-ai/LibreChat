# Application Sections

The interface is organized in an intuitive way, with a side menu for navigation and a main area for interaction. On the left is the chronologically organized conversation history, in the center is the chat area, and on the right are the additional settings.

![image](/help-images/en_0003.png)

All conversations remain saved and can be renamed if desired. Conversations are private for each user.

## Settings

General application settings are accessed by clicking the user name in the lower left part of the screen.

**Balance – Token Limit for Paid Agents**

This tab shows the token limit for each user. The limit is 20,000 tokens which reset daily at midnight; unused tokens are added to the next day’s total. A chat with history consumes more tokens than a new chat.

***Note: Only "Assistant with knowledge" or a user-created agent that selected the Claude model will consume tokens.***

## Vertical Menu

The vertical menu is positioned on the left side of the interface and displays the complete history of conversations, organized chronologically. This section offers quick access to all previous chats and allows efficient management of them.

![image](/help-images/en_0013.png)

If you want to share, rename, copy or delete a chat from the list, this can be done by pressing the three dots and selecting the desired option.

![image](/help-images/en_0014.png)

**Available Functions:**

**The “Share”**

Allows sharing the page and its content to other team members. A unique link is generated that can be copied and distributed.

![image](/help-images/en_0015.png)

![image](/help-images/en_0016.png)

The **“Rename”**

Changes the conversation name for faster identification in the list. Useful for thematic organization of discussions.

![image](/help-images/en_0017.png)

The **“Duplicate”**

Creates an exact copy of the chat screen, preserving the entire conversation history.

![image](/help-images/en_0018.png)

The **“Archive”**

Archives the conversation, removing it from the main list without permanently deleting it. Archived chats remain accessible through: Settings → General → Archived chats

![image](/help-images/en_0019.png)

![image](/help-images/en_0020.png)

From the archive management screen, you can:

Filter – Quick search through archived conversations

Unarchive – Restore chat to main list

Delete – Permanent removal from archive

![image](/help-images/en_0021.png)

The **“Delete”** allows permanent deletion of desired chat screens.

![image](/help-images/en_0022.png)

## Vertical Menu Buttons

At the top of the vertical menu are several essential buttons for interface control and conversation management. Each button offers specific functionalities that optimize the user experience.

**Close Sidebar** **– Ascundere Meniu Vertical**

Allows hiding the side menu to maximize workspace. Useful when you want to focus exclusively on the active conversation or when working on smaller screens.

![image](/help-images/en_0023.png)

**Agent Marketplace – Select Personal Agents**

From this button you can select the agents created by each user and the agent for time logging to PPM.

![image](/help-images/en_0024.png)

![image](/help-images/en_0025.png)

**Bookmarks – Quick Bookmarks**

Advanced functionality for saving keywords associated with important conversations. Bookmarks facilitate quick identification of specific chats in the vertical list, allowing easy resumption of discussions or quick access to relevant information and answers obtained previously.

![image](/help-images/en_0026.png)

Managing Bookmarks:

To mark a conversation: Add descriptive keywords

To delete all bookmarks: Press the button **"Clear all"**

![image](/help-images/en_0027.png)

**New Chat – New Conversation**

Opens a new chat window to initiate a separate conversation. Use this button to start a new work session, keeping conversations organized on distinct topics.

![image](/help-images/en_0028.png)

**Workspace – Personalised Workspaces**

Functionality that allows creating dedicated work environments, with personalized AI agents that respond exclusively based on specific uploaded documentation.

Use Cases:

Projects with specific clients who have proprietary documentation

Teams working on separate projects with dedicated resources

Need for contextualized responses based on specific materials

Workspace-ul se creeaza prin apasarea butonului Select Workspace > Create Workspace.

![image](/help-images/en_0029.png)

**Workspace Settings:**

From the configuration menu you can:

Manage team access – Add/remove members

Rename workspace – For easier identification

Add additional information – Descriptions, labels, notes

![image](/help-images/en_0030.png)

![image](/help-images/en_0031.png)

**Workspace Components:**

The workspace interface displays:

Active conversations – All chats from that space

Defined agents – List of configured AI assistants

Members – Users with access to the workspace

Uploaded files – Documentation available for analysis

![image](/help-images/en_0032.png)

**Upload Documentation:** To add new documentation, press the button **Upload** and select the desired files. The documents will be processed and integrated into the agent's knowledge base.

![image](/help-images/en_0033.png)

**Assistant Selector – Select Assistant**

Allows changing the active assistant in the current conversation. Each functionality offers varied capabilities:

Assistant with knowledge - answers questions based on documentation uploaded in Confluence.

File Search - answers questions based on documentation uploaded in chat

Assistant - ofoffers a series of functionalities: translations, generation of documentation and diagrams, code review

My agents provides access to agents created by the user.

![image](/help-images/en_0034.png)
